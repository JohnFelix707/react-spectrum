## Context

Context properties such as isDisabled, isEmphasized, isQuiet, isReadOnly,
isRequired, and validationState are passed into the provider as props and used
by the children components via a context.

### Disabled

```tsx example
<Provider isDisabled>
  <Checkbox>
    Checkbox Label
  </Checkbox>
  <Button variant="cta">Button</Button>
</Provider>
```

## Global Settings

These properties such as theme, locale, and scale adjust all children components.

### Theme and Scale

```tsx example
<Provider theme="dark" scale="large">
  <Checkbox>
    Checkbox Label
  </Checkbox>
  <Button variant="cta">Button</Button>
</Provider>
```

### Locale

Adjust the app locale from the browser value, which changes how all children
components are internationalized.

```tsx example
<Provider locale="he-IL">
  <Checkbox>
    Checkbox Label
  </Checkbox>
  <Button variant="cta">Button</Button>
</Provider>
```
