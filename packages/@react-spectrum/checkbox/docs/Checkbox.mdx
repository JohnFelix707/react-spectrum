## Content

Checkboxes can have a visual label. The Checkbox is placed on the left side of the text.

```jsx example
<Checkbox><span>Checkbox Label</span></Checkbox>
```

### Accessibility

If Checkboxes have a label then they are accessible by default.
If a visible label isn't specified, an `aria-label` should be provided to the Checkbox.

### Internationalization

To internationalize a Checkbox, a localized string should be passed to the label prop of the Checkbox.
For languages that are read right to left, Hebrew and Arabic for example, the Checkbox is placed on the right side of the text.

## Events

Checkboxes accept a user defined `onChange` event, triggered whenever the Checkbox is clicked.
The example below uses `onChange` to log how the user is interacting with the component.

```jsx example
function Example() {

  let toggle = (value) => {
    if (value) {
     alert('The Checkbox is checked');
    } else {
     alert('The Checkbox is unchecked');
    }
  }

  return (
    <Checkbox
      onChange={toggle}
      aria-label="This checkbox has no visible label"
    />
  );
 }
```

## Validation

Visually displaying a Checkbox's `valid` and `invalid` state is supported via the `validationState` prop.
Note that determining if a Checkbox is `valid` or `invalid` is up to the end user.

```jsx example
<Checkbox validationState="invalid" />
```

```jsx example
<Checkbox validationState="valid" />
```
