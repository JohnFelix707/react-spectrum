## Content

Text fields can render an icon inside itself, useful for visualizing the field's purpose.

```tsx example
<TextField icon={<Info />} aria-label="Info field" />
```

## Value

A text field's `value` is empty by default, but an initial, uncontrolled, value can be provided using the `defaultValue` prop.
Alternatively, providing a `value` prop creates a textfield with a controlled value. 

```tsx example
<TextField defaultValue="Uncontrolled Text" aria-label="uncontrolled textfield" UNSAFE_style={{'margin-right': '20px'}} />
<TextField value="Controlled Text" aria-label="controlled textfield" />
```

Placeholder text that describes the expected value or formatting for the field can be provided using the `placeholder` prop. 
Placeholder text will only appear when the text field is empty, and should not be used as a substitute for labelling the control with a visible label.

```tsx example
<TextField placeholder="abc@adobe.com" aria-label="email" />
```

## Labeling

A visual label should be provided for the text field using the `label` prop. If the field's required state should be displayed,
the `isRequired` and `necessityIndicator` props can be provided to the text field as well.

```tsx example
<TextField label="Street address" UNSAFE_style={{'margin-right': '20px'}} />
<TextField label="Street address" isRequired necessityIndicator="icon" UNSAFE_style={{'margin-right': '20px'}} />
<TextField label="Street address" necessityIndicator="label" />
```

### Accessibility

If a visible label isn't specified, an `aria-label` should be provided to the text field for 
accessibility. If the field is labeled by a separate element instead, an `aria-labelledby`
matching the labeling element's `id` should be provided instead.

### Internationalization

In order to internationalize a text field, a localized string should be passed to the `label` or `aria-label` prop.
A localized text string or alternative text is provided for the necessity indicator by default.

## Events

Text fields accept a user defined `onChange` event, triggered whenever the contents of the text field
is altered. Other events such as `onBlur` and `onFocus` are also accepted, see the `Props`
list at the bottom for a list of all supported events.

The example below uses `onChange` to validate if the user has entered a number larger than 15 into the 
textfield. 

```tsx example
function Example() {
  let [isValid, setValid] = React.useState(true);

  let validate = (value) => {
    setValid(Number(value) > 15)
  }

  return (
    <TextField
      validationState={isValid ? 'valid' : 'invalid'}
      placeholder="Enter a number"
      onChange={validate}
      aria-label="number field"
    />
  );
}
```

## Validation

Visually displaying a text field's `valid` and `invalid` state is supported via the `validationState` prop.
Note that the logic to determine what inputs are `valid` or `invalid` must be implemented by the user.

```tsx example
<TextField defaultValue="This is invalid" aria-label="textfield" validationState="invalid" UNSAFE_style={{'margin-right': '20px'}} />
<TextField defaultValue="This is valid" aria-label="textfield" validationState="valid" />
```
