## Content

Radio buttons can have a visible `label`, although they are not required to have one.
All radio components, even if there is only one, cannot be rendered outside of a radio button group.

## Value

Only one radio button per radio button group can be selected at a time.

### Accessibility

When the radio is checked, the appropriate ARIA state attribute will automatically be applied.

### Default Checked

A group of radio buttons can be rendered with an option already selected.
The Radio option that is checked by default can be picked by passing in this prop where the string value
matches the value prop passed to the radio button.

```tsx example
<RadioGroup label="Example Options" defaultValue="dragon">
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

### Controlled

Prevent users from toggling between different options.

```tsx example
<RadioGroup label="Example Options" value="dragon">
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

## Labeling

When a visible label isn't being used an `aria-label` prop is needed to explicitly set the accessibility name for the radio button group.
An `aria-label` prop should not be used for an individual radio button as it is they should always have a visible value.

```tsx example
<RadioGroup aria-label='Favorite Avatar'>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

### Internationalization

To internationalize a radio button, a localized string should be set as the child content of the Checkbox.
For languages that are read right to left (e.g. Hebrew and Arabic), the radio button is placed on the right side of the text.

## Events

Radio buttons accept a user defined `onChange` prop, triggered whenever the radio button is clicked.
The example below uses `onChange` to log how the user is interacting with the component.

```tsx example
function Example() {
  let [selected, setSelection] = React.useState('none');

  return (
    <div>
      <div>You have selected: {selected}</div>
      <RadioGroup aria-label="Favorite Avatar" onChange={value => setSelection(value)}>
       <Radio value="wizard">
         Wizard
       </Radio>
       <Radio value="dragon">
         Dragon
       </Radio>
      </RadioGroup>
    </div>
  );
}
```

## Validation

### Invalid

Visually displaying a radio button's `valid` and `invalid` state is supported via the `validationState` prop.
Note that the logic to determine what inputs are `valid` or `invalid` must be implemented by the user.

```tsx example
<RadioGroup label="Example Options" validationState='invalid'>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

## Visual Options

### Orientation

The default orientation is to be aligned vertically.
An optional prop can be passed to the radio button group to position the radio buttons inline.

```tsx example
<RadioGroup label="Example Options" orientation='horizontal'>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

### Label Position

The default position of a label is on top of the radio button group.
An optional prop can be passed to the radio button group that will position the label at the start of the topmost radio button.

```tsx example
<RadioGroup label="Example Options" labelPosition='side'>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

### Label Align

The default top position for a label is left aligned (right in rtl languages).
A label can be right aligned (left in rtl languages) via an optional prop to the radio button group.

```tsx example
<RadioGroup label="Example Options" labelAlign='end'>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

### Disabled

The disabled prop applies a disabled style as well as preventing click events and
keyboard navigation from performing selection.
This prop can be passed to the entire radio button group or individual radio buttons.
[View Guidelines](https://spectrum.adobe.com/page/radio-button/#Disabled)

```tsx example
<RadioGroup label="Example Options" isDisabled>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

```tsx example
<RadioGroup label="Example Options">
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon" isDisabled>
   Dragon
 </Radio>
</RadioGroup>
```

### Read Only

In this mode no visual indication is given to the user that any option within the radio button group has been selected.
This prop can be passed to the entire radio button group but not to individual radio buttons.

```tsx example
<RadioGroup label="Example Options" isReadOnly>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```

### Emphasized

By default, radio buttons are not emphasized (gray filling).
The emphasized style provides a blue filling for visual prominence.
[View Guidelines](https://spectrum.adobe.com/page/radio-button/#Emphasis)

```tsx example
<RadioGroup label="Example Options" isEmphasized>
 <Radio value="wizard">
   Wizard
 </Radio>
 <Radio value="dragon">
   Dragon
 </Radio>
</RadioGroup>
```
